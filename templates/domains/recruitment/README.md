# Recruitment Domain

This domain covers all aspects of the recruitment and hiring process, from candidate sourcing to onboarding.

## Domain Overview

The Recruitment domain manages the complete talent acquisition lifecycle, including:
- Candidate pipeline and stage management
- Application processing and candidate profiles
- Interview scheduling and assessment
- Offer management and acceptance
- Onboarding coordination
- Recruitment analytics and compliance

## Domain Expert Agents

This domain is broken down into specialized expert agents:

1. **Domain Expert — Recruitment (Pipeline & Stages)**
   - Focus: Workflow management, stage transitions, pipeline rules
   - Template: `domain_expert_recruitment_pipeline_stages.md`

2. **Domain Expert — Recruitment (Candidate Management)**
   - Focus: Candidate profiles, applications, sourcing, data management
   - Template: `domain_expert_recruitment_candidate_management.md`

3. **Domain Expert — Recruitment (Interview & Assessment)**
   - Focus: Interview scheduling, feedback collection, assessment scoring
   - Template: `domain_expert_recruitment_interview_assessment.md`

4. **Domain Expert — Recruitment (Offer & Onboarding)**
   - Focus: Offer creation, negotiation, acceptance, pre-boarding workflows
   - Template: `domain_expert_recruitment_offer_onboarding.md`

5. **Domain Expert — Recruitment (Analytics & Compliance)**
   - Focus: Recruitment metrics, reporting, legal compliance, diversity tracking
   - Template: `domain_expert_recruitment_analytics_compliance.md`

## Usage

When working on recruitment-related tickets:

1. Identify which subdomain(s) the ticket touches
2. Run the appropriate Domain Expert agent(s) using the template files
3. Collect the `DomainKnowledgePack` JSON output(s)
4. Use the pack(s) as input to Agent 02 (Product Analyst) for backlog generation
